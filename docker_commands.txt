Команды для докера:

docker pull *название образа из docker-hub (можно с тегом и т. д.)*
docker images - список образов
docker ps - список только запущенных контейнеров
docker ps 
          -a - список всех контейнеров
          -q - выведет только id всех контейнеров
docker start - запустит остановленный контейнер
docker exec *опции запуска: -it* *какое ПО использовать для действий внутри контейнера: bash* - зайти внутрь РАБОТАЮЩЕГО контейнера (например в БД)
docker run *название образа* - создаст и запустит контейнер из образа
    --rm - удалить контейнер после установки
    -d - запуск контейнера в фоне
    --name *придуманное имя контейнера* - дать контейнеру собственное имя
    -i - вход в контейнер после запуска в интерактивном режиме
    -it - вход в контейнер после запуска в интерактивном режиме + подключение терминала
docker stop *название (id) контейнера* - остановить контейнер
docker rm *название (id) контейнера* - удалить контейнер
docker rmi *название образа* - удалить образ
docker commit *существующий конейнер* *новый образ* - создание нового образа из существующего контейнера
docker stop $(docker ps -q) - остановка всех запущенных контейнеров по id
docker rm $(docker ps -qa) - удалит все существующие контейнеры
docker history *название образа* - вывести слои образа
> /dev/null - после docker run - не выведет id запущенного контейнера в консоль





Команды из курса:
Работа с образами
docker images или docker image ls — посмотреть список образов (ссылка, ссылка)

docker rmi <образ> [образ...] или docker image rm <образ> [образ...] — удалить образ(ы) (ссылка, ссылка)

Работа с контейнерами
docker run <образ> — поднять контейнер на основе образа (ссылка)

      docker run --name <имя> <образ> — при поднятии присвоить имя контейнеру (ссылка)

      docker run --rm <образ> — удалять контейнер после завершения его работы (ссылка)

      docker run -it <образ> — позволяет «войти» в контейнер во время его создания (ссылка, ссылка)

      docker run -d <образ> — поднять контейнер в фоновом режиме (ссылка)

docker ps — список активных (работающих) контейнеров (ссылка)

      docker ps -a — список всех контейнеров (ссылка)

docker stop <контейнер> [контейнер...] — остановить работающий(ие) контейнер(ы) (ссылка)

docker start <контейнер> [контейнер...] — запустить остановленный(ые) контейнер(ы) (ссылка)

docker rm <контейнер> [контейнер...] — удалить контейнер(ы) (ссылка)

docker exec <контейнер> команда — запустить команду в работающем контейнер (ссылка)

      docker exec -it <контейнер> bash — запустить bash процесс и «войти» в контейнер (ссылка)
